# 标签管理

只有在下面两种情况下会创建新标签页：

1. 在漫画列表页，点击某本漫画，进入漫画详情页，此时生成该漫画的标签页；如果在漫画详情页，继续阅读，进入阅读模式，此时标签页不变
2. 在书籍的列表页，继续阅读，进入阅读模式，此时生成该书的标签页

标签页的作用：

标签页能够保存状态，点击标签页，能够恢复上次浏览时的状态：

1. 漫画阅读页和书籍阅读页，依旧显示上次的阅读进度；
2. 漫画详情页，依旧展开所有漫画图片与滚动浏览的位置；

标签页的特殊逻辑：

1. 对于同一个路径，永远只存在一个标签页，后面创建的新标签页覆盖掉旧的标签页

## 标签页切换优化【重点！！】

我可能打开10本漫画，创建10个标签页，每本漫画200张图片，现在已经使用了虚拟DOM，但是还需要优化。

此时切换标签页的流畅度就非常重要，所以我提出以下优化方案。

方案：标签页保活 + 多级缓存 + LRU + 按活跃度释放

核心要求是实现类似谷歌浏览器不同标签页的流畅切换。
